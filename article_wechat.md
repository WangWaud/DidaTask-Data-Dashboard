# 用 AI 帮我"审判"了自己的时间

> 不是在用滴答清单，是在被滴答清单记录着。

---

## 起因：时间管理

长期以来，我一直在使用滴答清单（高级会员版）来记录我的日程安排，它在多个项目并行进行的时候特别的好用。

![](https://cdn.jsdelivr.net/gh/WangWaud/Images/img/202603011039834.png)

在此基础上，我在次周都会进行周总结，来看看上周的目标是否有完成，上周的工作时间是否够量，再直白一点就是：

**我这周到底把时间花在哪儿了？**

文献读了多久？实验占了几小时？还是说大部分时间其实在"休闲娱乐"里悄悄溜走了……

滴答清单本身没有这个统计视图，我也咨询过官方团队，团队表示暂时也没有开发需求。

![](https://cdn.jsdelivr.net/gh/WangWaud/Images/img/202603011043666.png)

于是我决定自己搞一个。目标很简单：**一个能读取我自己任务数据、展示时间分布的本地看板。**

![](https://cdn.jsdelivr.net/gh/WangWaud/Images/img/202602281707132.png)

---

我希望借助AI能够在短时间内更加高效的获取时间分布信息。首先尝试了直接将清单周视图截图发给Gemini识别，时间信息的识别准确性很低，因此放弃了这个方案。

## 难点：听起来简单，做起来全是坑

### 坑一：官方 API 不返回已完成任务

一开始想走正规路子，用滴答清单的 Open API 拉数据。结果发现——**v1 接口压根不返回已完成的任务。**

已完成任务才是我最想分析的东西啊！这条路堵死了。

最后的解决方案：改用 **CSV 备份导入**。**滴答清单网页版**支持导出完整的备份文件，包含所有历史任务，字段齐全。

### 坑二：CSV 里有多行文本字段，split 一刀切直接炸了

任务的"备注"字段里如果有换行符，CSV 文件里对应的那条记录就会跨多行。  
最开始用 `split('\n')` 直接切，遇到这种任务就直接解析错位、数据全乱。

后来重写了一个**字符流解析器**，逐字符处理，正确识别带引号的多行字段，才彻底解决。

### 坑三：清单名带 Emoji，颜色全变灰

我的清单名都带了 Emoji，比如"📄文献"、"⚗️实验"。  
代码里配色表的 key 是"文献"、"实验"，匹配不上，所有图表一片灰色。

解决方案：写了个 `stripEmoji()` 函数，用 Unicode 正则把开头的 Emoji 剥掉再查表。  
但又发现 ⚖️（U+2696 + U+FE0F 变体选择符）还是没被正确去掉……最后把正则扩展到覆盖所有 Emoji 相关的 Unicode 属性才搞定。

### 坑四：时间统计差了好几分钟

图表显示 3h18m，但表格里两个任务加起来明明只有 3h15m。  

查了半天发现：数据传给 ECharts 之前做了一次 `Math.round(hours * 10) / 10` 的四舍五入（精度 0.1h = 6分钟）。  
3.25h 被四舍五入成 3.3h，3.3 × 60 = 198 分钟，就成了 3h18m。  
把那行多余的 round 去掉就好了——原来错在「多此一举」。

## 我是怎么做到的：VSCode + GitHub Copilot

说实话，这个项目我**完全没有手写代码**。

我的工作流是这样的：

1. **描述需求** → 在 VSCode 里直接跟 Copilot 对话，说我想要什么功能
2. **Copilot 生成代码** → 直接写进对应文件
3. **我来测试** → 刷新页面，看效果，发现问题
4. **描述 bug** → 告诉 Copilot 现象（比如"图表显示3h18m但实际是3h15m"），它来定位和修复
5. **反复迭代** → 功能一点点完善

整个过程更像是**我作为产品经理 + 测试工程师，Copilot 作为开发工程师**。

我需要做的事情：
- 清楚地描述我想要什么
- 能识别结果对不对
- 知道哪里不对、大概是什么方向的问题

我不需要做的事情：
- 记住 ECharts API 怎么写
- 自己想 CSV 解析的边界情况
- 手动调试 Unicode 正则

> 有一说一，Copilot 有时候也会犯低级错误或者绕弯路，但告诉它哪里不对之后，它修得很快。

## 怎么用这个项目

项目已开源：**[github.com/WangWaud/DidaTask-Data-Dashboard](https://github.com/WangWaud/DidaTask-Data-Dashboard)**

### 第一步：把项目跑起来

需要提前安装好 [Node.js](https://nodejs.org/)（v16+ 即可）。

```bash
# 1. 克隆项目
git clone https://github.com/WangWaud/DidaTaskManager.git
cd DidaTaskManager

# 2. 安装依赖
npm install

# 3. 启动
npm start
```

打开浏览器访问 `http://localhost:3000`，看板就出来了。

### 第二步：导出你的滴答清单数据

1. 登录 [dida365.com](https://dida365.com)（网页版）
2. 左上角头像 → **设置** → **账户** → **数据备份**
3. 点击「生成备份」，下载 `.csv` 文件

### 第三步：导入数据

在看板页面右上角点击 **📂 导入 CSV**，选择刚才下载的文件，数据秒速加载。

> 数据会缓存在浏览器的 localStorage 里，下次打开不用重新导入。

### 使用效果

- **日 / 周 / 月 / 季度 / 年** 五个时间维度随意切换
- 右上角的日期选择器可以直接跳转到某一天
- 点击清单名称或标签可以单独筛选
- 任务名称太长？鼠标悬停会展开显示完整内容

## 如何用 AI 给自己加功能

这个项目是按我自己的清单结构定制的，你的清单结构肯定和我不一样。  
好消息是：**你完全可以让 AI 帮你改成自己的版本**，不需要懂代码。

### 修改清单颜色和结构

打开 `public/app.js`，顶部有三个配置块：

```javascript
// 清单 → 文件夹 映射
const FOLDER_MAP = { ... };

// 清单颜色
const PROJECT_COLORS = { ... };

// 文件夹颜色  
const FOLDER_COLORS = { ... };
```

把这段代码复制给 AI（Copilot、Claude、ChatGPT 都行），然后说：

> *"我的滴答清单结构是这样的：[描述你的文件夹和清单]，帮我修改这三个配置。颜色我想要 [你的偏好]。"*

### 加新功能的示例 Prompt

**想加一个"本周 vs 上周"的对比图？**
> "在现有柱状图下面，增加一个本周与上周各清单时长对比的条形图，数据来源和现有图表一致。"

**想给某个清单单独设置目标时长并显示完成率？**
> "我想给'文献'清单设置每周目标是 10 小时，在清单筛选按钮旁边显示本周已完成的百分比。"

**想导出当前视图为图片？**
> "在看板右上角加一个'导出图片'按钮，点击后把图表区域截图保存为 PNG。"

AI 不会帮你做所有决定，但只要你描述清楚想要什么，它能把代码写出来。

## 多说两句：关于「vibe coding」时代的个性化开发

最近"vibe coding"这个词越来越流行——大意是说，你只需要描述感觉和需求，AI 负责把代码写出来。这个项目大概就是一个典型案例。

我觉得这个趋势是真实的，而且会越来越普及。**以后每个人都可以给自己造工具**，就像以前每个人都可以用 Excel 做表格一样——门槛会低到"会提问就会开发"。

但我也想泼一点冷水。

**开发的目的，是解决问题，而不是开发本身。**

这个看板之所以值得花时间做，是因为我真的需要这个数据、真的在用它做决策。如果只是"哇 AI 能帮我写代码好神奇"然后做了一堆没人用的东西，那不叫提效，叫换了一种方式浪费时间。

在动手之前，值得问自己一个问题：**"如果这个工具做出来了，我真的会用它吗？"**

如果答案是"会"，那就做。AI 已经把门槛降得足够低了。

---

所以，我的建议是：

**找一个你真实遇到的、现有工具解决不好的问题，然后去做。**

不需要会编程，不需要懂架构，只需要能把问题描述清楚。剩下的，AI 会帮你搞定大部分。

这是我觉得 vibe coding 真正有意义的地方——**它让"有想法但不会写代码"不再是一个借口**。
